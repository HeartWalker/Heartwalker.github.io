JS事件中target与currentTarget

By Heart Walker

發表於 2016-03-21

先来一段网上流行的解释

target是事件的调用对象(event dispatcher)，currentTarget是事件的处理对象(event processor)

target在事件流的目标阶段；currentTarget在事件流的捕获，目标及冒泡阶段。只有当事件流处在目标阶段的时候，两个的指向才是一样的，而当处于捕获和冒泡阶段的时候，target指向被单击的对象而currentTarget指向当前事件活动的对象（一般为父级）。

是不是有点晦涩,看了以后还是似懂非懂,一般网上的案例也是难以明白,于是我又专门测试了一下,然后用通俗的语言总结起来

下边是测试在控制台的结果

注:测试环境为当前最新版的firefox与chrome浏览器

总结:

  1. currentTarget就是绑定事件的节点,哪个节点绑定事件就是哪个节点
  2. currentTarget与this总是相同的
  3. target是最终触发事件的节点
  4. currentTarget与target有时相同有时不同
  5. 只有当绑定事件的节点对象是最终触发事件的节点对象时currentTarget才等于target

用js在html中写入内容

By Heart Walker

發表於 2016-03-13

document.write(), innerHTML, innerText, value, nodevalue

document.write

document.write是直接写入到页面的内容流，如果在写之前没有调用document.open, 浏览器会自动调用open。每次写完关闭之后重新调用该函数，会导致页面被重写。即第一次调用document.write会将当前页面的内容清空, 其后再次调用会在原来的内容上继续添加, 如果之后调用了document.close()或将页面关闭, 再次使用document.write会再次将页面内容清空并输出, 识别标签

innerHTML

innerHTML是dom元素的一个属性，代表该元素的html内容。你可以精确到某一个具体的元素来进行更改。如果想修改document的内容，则需要修改document.documentElement.innerElement, 识别标签

innerText

innerText设置的内容不识别标签, 获取内容是从起始位置到终止位置的内容, 但它去除Html标签, innerText并非W3C标准属性, 原为ie特有属性,经测试到目前为止已经得到主流浏览器最新版本的支持

value

用来设置和获取具有value属性标签的value值, 主要用于表单中

nodevalue

对于文本节点，nodeValue 属性包含文本。
对于属性节点，nodeValue 属性包含属性值。
nodeValue 属性对于文档节点和元素节点是不可用的。
nodeValue是W3C标准属性

link与@import引入css

By Heart Walker

發表於 2016-03-08

在写html页面时css的书写方式有行内式, 内嵌式和外联式三种; 实际开发中主要使用外联式与少量的内嵌式. 其中外联式有两种方式:link和@import.

语法

区别

  1. link属于XHTML标签，而@import完全是CSS提供的一种方式。 link标签除了可以加载CSS外，还可以做很多其它的事情，比如定义RSS，定义rel连接属性等，@import就只能加载CSS了。
  2. 加载顺序的差别。当一个页面被加载的时候（就是被浏览者浏览的时候），link引用的CSS会同时被加载，而@import引用的CSS会等到页面全部被下载完再被加载。所以有时候浏览@import加载CSS的页面时开始会没有样式（就是闪烁），网速慢的时候还挺明显。
  3. 兼容性的差别。由于@import是CSS2.1提出的所以老的浏览器不支持，@import只有在IE5以上的才能识别，而link标签无此问题。
  4. 使用dom控制样式时的差别。当使用javascript控制dom去改变样式的时候，只能使用link标签，因为@import不是dom可以控制的。
  5. @import可以在css中再次引入其他样式表，比如可以创建一个主样式表，在主样式表中再引入其他的样式表

js小数精度

By Heart Walker

發表於 2016-02-29

示例

在js中0.1+0.2结果是多少?0.3-0.2呢?下边是在chrome控制台的输出结果:

console.log(0.1+0.2)
0.30000000000000004
console.log(0.3 - 0.2)
0.09999999999999998

为什么不是0.3与0.1呢?

原因

Javascript采用了IEEE-745浮点数表示法（几乎所有的编程语言都采用），
这是一种二进制表示法，可以精确地表示分数，比如1/2，1/8，1/1024。
遗憾的是我们常用的分数（特别是在金融的计算方面）都是十进制分数1/10，1/100等。
二进制浮点数表示法并不能精确的表示类似0.1这样 的简单的数字。---------以上来自《Javascript权威指南P37》

那么让我们看一下二进制中0.1与0.2到底是多少

0.1 => 0.0001 1001 1001 1001…（无限循环）
0.2 => 0.0011 0011 0011 0011…（无限循环）

但是在js中小数位数是有限的:

console.log(Number(0.1).toString(2))
0.0001100110011001100110011001100110011001100110011001101

console.log(Number(0.2).toString(2))
0.001100110011001100110011001100110011001100110011001101

将他们的结果相加，我们再把它转换为十进制，就成了 0.30000000000000004。

z-index层级

By Heart Walker

發表於 2016-02-28

先说一下概念

因为显示器显示的图案是一个二维平面，拥有x轴和y轴来表示位置属性。为了表示三维立体的概念如显示元素的上下层的叠加顺序引入了z-index属性来表示z轴的区别，表示一个元素在叠加顺序上的上下立体关系。

注释： 所有主流浏览器都支持 z-index 属性。任何的版本的 Internet Explorer （包括 IE8）都不支持属性值 “inherit”。

IE6/7下position不为static，且z-index不存在时z-index为0，除此之外的浏览器z-index为auto。

z-index为auto的元素不参与层级关系的比较，由向上遍历至此且z-index不为auto的元素来参与比较。

z-index 属性设置元素的堆叠顺序。拥有更高堆叠顺序的元素总是会处于堆叠顺序较低的元素的前面。z-index 仅能在定位元素上奏效。
注释：元素可拥有负的 z-index 属性值。

一些总结

关于z-index的介绍网络上有许多,为了能简单快速的学会使用层级,我在这里总结了一些效果如下:

1. z-index仅在定位中起作用
2. 正常情况浮动层级大于标准流 ,定位流大于浮动流 ,值为auto, 但它们都在0到1之间
3. 如果父子元素都有z-index值,不论正负,那么以父盒子为准,不管父子的z-index是多少
所以如果两个此时盒子是兄弟盒子并且都有定位的z-index值, 那么它们子盒子的层级由父盒子决定
4. 如果父盒子没有定位或只定位没有写z-index值,父盒子的的z-index是auto,  此时会有两种情况
a:如果给子盒子一个正的z-index值,此时子盒子的z-index起作用
b:如果给子盒子一个负的z-index值,此时子盒子会在父盒子之下

border-image-width

By Heart Walker

發表於 2016-02-25

border-image-width

border-image是css3新增的一个属性,它允许我们以九宫格的方式将图片切割并且运用到边框中,其中border-image-width决定了切割后再边框上显示的图片的大小

border-width与border-image-width

那么它们有什么关联呢?
首先border-image-width本身并不占位置,如果盒子有边框那么border-image会以border最外侧为边界开始,如果没有border-width那么border-image会以padding甚至内容的边界开始渲染;也就是说border-width影响border-image但并不决定它

border-image-width的兼容性问题

经过我的测试:在FF下如果没有border-width对盒子没有任何影响;但是如果在chrome下会自动渲染出3px的border-width,可以手动设置border-width的宽度但要保证至少一边的border-width不为零,否则border-image就会消失

总结

若果要使用border-image最好在border上使用:即要有border-width属性
